Consolidation-Review Skill 架构

Python脚本 = 稳定引擎（标准逻辑）
Skill审阅层 = 灵活适配（应对标准逻辑外的特殊情况，AI检查、AI思考、AI修正）

Prompt 1：你做出来的这些python脚本，如果某一期我给你的科目余额表里科目有些变化，你做出来的结果不对了，按照现在你的逻辑会改python脚本，但是这个调整也许只适用于这一期，而你改掉了整个脚本，后续做下一期的时候又会做不对了。你现在的模式没有灵活性。你觉得是不是做成一个skill，你用python跑完之后，skill来补充一些灵活性，你跑完python后直接去检查跑出来的表，如果有任何check出现了问题，你可以检查并调整你的逻辑，但只是一次性的调整，不动到所有python。

  5. step_D_notes.py - 附注
  6. Step_E_sheets.py - SEC披露附表
3. 公式计算
macOS open Excel → AppleScript保存触发计算
4. 自动审阅
review_checks.py 扫描所有含"check"的行
7. 记录固化
记录到memory
6. 修补处理
patch_cell.py 修补（或确认为正常差异）
5. 诊断分析
根因定位：数据问题 / 脚本bug / 一次性特殊

当前存在的问题

  1. 在本地版大模型不够强的情况下，训练是依靠外部模型，所有涉及到的数据和表格都需要脱敏处理，花费大量时间
